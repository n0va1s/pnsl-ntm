O SFC deve:

**PHP section (new class extends Component):**
- `public $equipes;` — coleção pública
- `mount()`: `$this->authorize('viewAny', Equipe::class);` + `$this->carregarEquipes();`
- `carregarEquipes()`: query `Equipe::paraMovimento(Auth::user()->idt_movimento)->orderBy('nom_equipe')->get()` — exibe apenas equipes ativas (sem withTrashed) per decisão resolved Q2 do planning context: "Index mostra apenas equipes ativas por padrão"
- `arquivar(int $idtEquipe)`: `$equipe = Equipe::findOrFail($idtEquipe); $this->authorize('update', $equipe); $equipe->delete(); $this->carregarEquipes(); session()->flash('success', 'Equipe arquivada com sucesso.');`
- `restaurar(int $idtEquipe)`: `$equipe = Equipe::withTrashed()->findOrFail($idtEquipe); $this->authorize('update', $equipe); $equipe->restore(); $this->carregarEquipes(); session()->flash('success', 'Equipe restaurada com sucesso.');`

**Nota sobre restaurar:** Para que a ação restaurar seja testável no Volt::test(), o index deve chamá-la com withTrashed. O RESEARCH.md (Pattern 3) nota que withTrashed() no carregarEquipes() exibiria arquivadas para coord-geral — mas resolved Q2 diz "apenas ativas". A restauração pode ser feita via um botão que aparece em uma lista separada ou via parâmetro URL. Para Phase 3: carregarEquipes() carrega ativas; restaurar() é action pública que recebe o idt_equipe (funciona mesmo que a equipe não esteja na lista exibida).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL params → route model binding | {equipe} na URL resolve para Equipe via idt_equipe — precisa de binding correto |
| Wire action → arquivar/restaurar | Usuário pode invocar arquivar(N) com qualquer N via wire:click — precisa de authorize() no action |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Elevation of Privilege | routes/web.php — equipes.create | mitigate | ->can('create', Equipe::class) bloqueia na camada de rota; $this->authorize() no mount() do SFC é segunda camada |
| T-03-06 | Elevation of Privilege | routes/web.php — equipes.edit | mitigate | ->can('update', 'equipe') com route model binding; $this->authorize('update', $equipe) no mount() |
| T-03-07 | Elevation of Privilege | equipes.index — arquivar(int $idtEquipe) | mitigate | $this->authorize('update', $equipe) dentro do action — usuário não pode arquivar equipe de outro movimento porque o Gate consulta EquipePolicy::update() que verifica papel na pivot |
| T-03-08 | Tampering | IDOR via idt_equipe em arquivar() | mitigate | Equipe::findOrFail() carrega o model real; authorize() valida a permissão sobre aquele model específico |
| T-03-09 | Denial of Service | wire:confirm em arquivar | accept | Confirmação client-side previne clique acidental; não é proteção de segurança (pode ser bypassada via wire:call direto — mitigado pelo authorize() server-side) |
